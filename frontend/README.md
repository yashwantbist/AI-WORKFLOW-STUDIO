# Frontend — README

> **Status:** Not yet implemented. This document describes the planned architecture.

---

## Overview

The frontend is the user-facing interface for NVIDIA AI Workflow Studio. It provides:

1. **Authentication screens** — sign-up, login, and logout flows.
2. **Document manager** — upload, list, and delete personal documents.
3. **Chat interface** — a conversational Q&A interface that displays the AI answer alongside source citations.
4. **Transparency panel** — shows which document chunks were retrieved and used to generate each answer.
5. **Disclaimer UI** — persistent notice that answers are AI-generated and may contain errors.

---

## Planned Technology Stack

| Component | Planned Technology |
|---|---|
| Framework | React (TypeScript) |
| UI library | shadcn/ui + Tailwind CSS |
| State management | React Query (server state) + Zustand (local state) |
| API communication | Fetch API with JWT in Authorization header |
| Bundler | Vite |

## Environment Variables

The frontend reads the following environment variables at build time (via Vite's `VITE_` prefix convention):

```
VITE_API_BASE_URL=http://localhost:8000
```

**No API keys or secrets belong in the frontend build.** All sensitive operations are performed by the backend.

---

## Running Locally (Planned)

```bash
npm install
npm run dev
```

The development server will start at `http://localhost:5173` and proxy API requests to the backend.

## UX Safety Requirements

- Every AI-generated answer must display a visible disclaimer: *"This answer was generated by an AI model. Verify important information against the source documents shown below."*
- Source citations (document name, page/section) must be displayed alongside every answer.
- The "Delete my documents" action must require an explicit confirmation step.
- Session tokens must be stored in memory (not `localStorage`) to reduce XSS exposure.
- See [`../docs/system-card.md`](../docs/system-card.md) for the full list of transparency requirements.
